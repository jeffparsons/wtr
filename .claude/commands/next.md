Pick a GitHub issue to work on and plan the work.

1. If an issue number was provided as an argument ("$ARGUMENTS"), use that. Otherwise, run `scripts/find-next-issue` to pick the highest-priority open issue.

2. Read the full issue body and all comments using `scripts/gh-issue-view <number>`. Understand the full scope of the work.

3. Enter plan mode and plan the implementation. The plan should include closing the GitHub issue after committing the work. Ask questions if anything is unclear or you suspect the user might have a preference that isn't already made explicit in the issue.
